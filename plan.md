# BIT39기 주니어 산학활동 총정리 포트폴리오 사이트 제작 계획

## Goal

박지원의 한 학기 BIT 활동(BIP, CP01, CP02)을 인턴 채용 담당자가 빠르게 이해할 수 있고, 동시에 본인이 학기 활동을 다시 돌아볼 수 있는 HTML 포트폴리오/아카이브 사이트로 구성한다. 단순 프로젝트 나열이 아니라, `문제를 구조화하고 실행 가능한 전략으로 바꾸는 연습을 했다`는 성장 서사가 첫 화면부터 읽히도록 만든다.

## Audience

- 1차 독자: 박지원 본인, 인턴 채용 담당자
- 보조 독자: 면접관, 자기소개서/포트폴리오를 검토하는 실무자
- 원하는 인상: 깔끔하고 전문적이며, 전략기획 직무에 맞는 문제정의/분석/실행화 역량이 드러나는 사람

## Content

사용자 기본 정보:

- 이름: 박지원
- 전공/학년: 경영학과 3학년
- 희망 직무: 전략기획
- 전체 성장 문장: 문제를 구조화하고 실행 가능한 전략으로 바꾸는 연습을 했다.

포함할 프로젝트:

1. BIP: 에브리타임의 DAU 감소 원인을 AI·SNS·유사 커뮤니티 이탈과 트래픽 불균형으로 구조화하고, 알고리즘 기반 필터링과 생활형 기능 전략을 제안
2. CP01: K리그를 국내 프로스포츠와 글로벌 축구 리그의 교집합에서 분석하고, 관람 효용 증진과 장벽 저하를 동시에 달성하기 위한 사진패스·응원부스 전략 제안
3. CP02: PM으로서 국내 체류 외국인의 체류 여정을 구조화하고, 케이뱅크의 초기 유입 및 지속 방문 전략 흐름을 설계

상세 페이지에는 아래 파일에 정리된 내용이 모두 들어가야 한다.

- `BIP_박지원_기여정리.md`
- `CP01_박지원_기여정리.md`
- `CP02_박지원_기여정리.md`

## Structure

사이트는 정적 HTML 파일 여러 개로 만든다.

1. `index.html`
   - Hero: `BIT39기 주니어 산학활동 총정리`
   - 박지원 / 경영학과 3학년 / 전략기획
   - 전체 성장 문장
   - 한 학기 활동 흐름 요약: BIP → CP01 → CP02
   - 역량 요약: 문제 구조화, 시장/고객 분석, 전략 구체화, PM 리딩
   - 프로젝트 카드 3개: 클릭하면 각 상세 페이지로 이동

2. `bip.html`
   - BIP 프로젝트 요약
   - 문제정의, 핵심 인사이트, 최종 전략
   - 노션 작업과 최종 발표 반영 내용 대조 표
   - 박지원 역할 및 기여
   - 포트폴리오용 문장
   - 면접 답변 초안

3. `cp01.html`
   - CP01 프로젝트 요약
   - 문제정의, 핵심 인사이트, 최종 전략
   - 노션 작업과 최종 발표 반영 내용 대조 표
   - 박지원 역할 및 기여
   - 포트폴리오용 문장
   - 면접 답변 초안

4. `cp02.html`
   - CP02 프로젝트 요약
   - 문제정의, 핵심 인사이트, 최종 전략
   - 노션 작업과 중간/최종 발표 반영 내용 대조 표
   - 박지원 역할 및 기여
   - 포트폴리오용 문장
   - 면접 답변 초안

공통 네비게이션:

- 홈
- BIP
- CP01
- CP02
- 각 상세 페이지 하단에 `다음 활동 보기`와 `전체로 돌아가기` 버튼

## Visual Direction

- 톤: 깔끔하고 전문적인 전략 포트폴리오
- 레이아웃: 이력서/컨설팅 리포트 느낌의 정보 밀도 있는 구성
- 색상: 화이트/오프화이트 배경, 딥 네이비 또는 차콜 텍스트, 포인트 컬러는 차분한 블루/그린 계열
- 타이포그래피: 시스템 산세리프 기반, 큰 제목은 단정하게, 본문은 읽기 편하게
- 인터랙션: 프로젝트 카드 hover, 상세 페이지 내 섹션 anchor, 모바일에서도 쉽게 읽히는 반응형 구성
- 아이콘/장식: 과한 장식 없이 작은 라벨, 단계 번호, 키워드 칩 중심

## Files

생성 예정 파일:

- `index.html`
- `bip.html`
- `cp01.html`
- `cp02.html`
- `style.css`

외부 라이브러리는 사용하지 않고, 로컬에서 바로 열 수 있는 정적 HTML/CSS로 제작한다.

## Assumptions

- 연락처, 이메일, 프로필 사진은 아직 제공되지 않았으므로 넣지 않는다.
- 사이트는 한국어로 제작한다.
- 각 상세 페이지의 본문은 기존 md 파일 내용을 빠짐없이 HTML 구조로 변환하되, 화면에서 읽기 좋게 섹션/표/카드 형태로 정리한다.
- PDF나 PPT 원본 링크는 사이트에 직접 첨부하지 않고, 현재 정리된 기여 내용 중심으로 구성한다.
