# 랜딩페이지 운영안

## 1. 목적

랜딩페이지는 제품 설명서가 아니라 "초기 수요 검증 장치"로 운영한다. 목표는 방문자가 20초 안에 서비스를 이해하고, 2분 안에 베타 신청 또는 파일럿 문의를 남기게 하는 것이다.

## 2. 우선 검증할 질문

- 사람들은 "내 필기체를 AI로 다시 쓰는 경험"에 돈을 낼 만큼 관심이 있는가?
- 개인 선물, 크리에이터 도구, 기관 체험 중 어떤 메시지가 가장 잘 전환되는가?
- 사용자가 기꺼이 제공할 수 있는 필기 샘플의 양은 어느 정도인가?
- 변형도 조절 기능이 차별점으로 이해되는가?
- 데이터 권리/삭제/악용 방지 설명이 신뢰를 높이는가, 혹은 전환을 방해하는가?

## 3. 페이지 구조

### 1. Hero

목표: 첫 화면에서 결과물과 감정적 사용처를 동시에 보여준다.

- 헤드라인 후보 A: "타이핑한 문장을, 내 손글씨처럼."
- 헤드라인 후보 B: "내 필체가 사라지지 않도록, AI로 다시 쓰세요."
- 서브카피: "손글씨 샘플을 학습해 편지, 카드, 다이어리, 초대장에 쓸 수 있는 개인 필기체를 만들어드립니다."
- CTA: "베타 신청하기", "기관 파일럿 문의"
- 필수 시각자료: 같은 문장을 변형도 0/50/100으로 출력한 비교 이미지.

### 2. 데모 섹션

목표: 텍스트 입력 → 손글씨 결과 → 변형도 조절을 한눈에 이해시킨다.

- 입력 예시: "고마워. 네가 남긴 말들이 오래 남았어."
- 결과 예시 3개: 정돈된 필체, 자연스러운 필체, 감성적 흔들림이 있는 필체.
- 컨트롤 예시: 정돈도, 자연스러운 변형, 잉크/종이 느낌, 문장 길이.

### 3. 사용처 섹션

카드형으로 4개만 제시한다.

| 사용처 | 카피 |
|---|---|
| 편지/카드 | "긴 글은 타이핑하고, 마지막 감정은 손글씨로." |
| 다이어리/문구 | "내 필체로 만드는 디지털 스티커와 메모." |
| 가족 기록 | "소중한 사람의 글씨를 오래 남기는 방법." |
| 브랜드 메시지 | "고객에게 더 사람다운 초대장과 감사장을." |

### 4. 작동 방식

너무 기술적으로 설명하지 않고 3단계로 끝낸다.

1. 손글씨 샘플을 업로드하거나 가이드 문장을 작성한다.
2. 필체의 획, 자간, 기울기, 리듬을 학습한다.
3. 원하는 문장을 입력하고 변형도를 조절해 내보낸다.

### 5. 신뢰/권리 섹션

랜딩페이지에서 반드시 보여줘야 한다.

- 업로드한 손글씨 데이터는 사용자의 소유다.
- 사용자는 언제든 삭제를 요청할 수 있다.
- 서명, 계약서, 공문, 사칭 목적 사용은 금지한다.
- 상업 이용은 별도 라이선스로 관리한다.

### 6. 신청 폼

폼은 짧게 시작하고, 제출 후 인터뷰 유도 질문을 받는다.

필수 입력:

- 이름/닉네임
- 이메일 또는 연락처
- 가장 쓰고 싶은 용도: 편지, 카드, 다이어리, 굿즈, 브랜드, 교육/기관, 기타
- 본인 손글씨 샘플 보유 여부
- 베타 참여 의향

선택 입력:

- 만들고 싶은 문장 1개
- 우려되는 점
- 인터뷰 가능 여부

## 4. A/B 테스트 운영

| 테스트 | A안 | B안 | 판단 기준 |
|---|---|---|---|
| 포지셔닝 | 감성 선물형 | 크리에이터 도구형 | 신청 전환율, 폼의 사용처 선택 |
| CTA | 베타 신청 | 내 글씨체 만들기 | 클릭률, 폼 완료율 |
| 데모 | 결과 이미지 중심 | 조절 UI 중심 | 스크롤 깊이, 데모 클릭 |
| 신뢰 문구 | 상단 노출 | 하단 노출 | 이탈률, 폼 완료율 |

## 5. 유입 채널

초기에는 넓은 광고보다 수요가 뾰족한 커뮤니티와 콘텐츠를 우선한다.

- 인스타그램: 손글씨 전후 비교, 편지/문구 감성 콘텐츠.
- 네이버 블로그/카페: 다이어리, 문구, 캘리그라피, 손편지 키워드.
- X/Threads: AI 데모형 영상, 개발/창작자 반응 수집.
- 크리에이터 협업: 캘리그라피 작가, 문구 작가, 다이어리 계정.
- 기관 아웃리치: 한글/편지쓰기/디자인/교육 관련 단체 파일럿 문의.

## 6. 핵심 지표

| 단계 | 지표 | 목표 기준 |
|---|---|---|
| 방문 | 세션, 유입 채널, 체류시간 | 채널별 비교 가능 수준 |
| 관심 | 스크롤 50% 이상, 데모 클릭 | 방문자의 30% 이상 |
| 전환 | 베타 신청 폼 완료 | 초기 5~10%면 긍정 신호 |
| 품질 | 인터뷰 동의율, 샘플 업로드 의향 | 신청자의 30% 이상 |
| 세그먼트 | 사용처 분포 | 1~2개 핵심 타깃 식별 |

## 7. 제작 체크리스트

- 실제 결과물처럼 보이는 데모 이미지 5종.
- 모바일 첫 화면에서 헤드라인, 결과 이미지, CTA가 모두 보이는 구조.
- 베타 신청과 기관 문의 CTA 분리.
- 데이터 권리/삭제/악용 금지 문구.
- GA4 또는 Plausible, Meta Pixel, UTM 체계.
- 신청자 자동 응답 메일: "샘플 안내", "인터뷰 요청", "출시 알림".

## 8. 2주 운영 루틴

- 매일: 신청자 확인, 유입 채널별 전환율 기록, DM/문의 응답.
- 주 2회: 랜딩 카피와 데모 이미지 교체 테스트.
- 주 1회: 신청자 5명 인터뷰, 전환 데이터 회고.
- 2주 후: 가장 강한 세그먼트 1개를 골라 다음 랜딩페이지 버전의 첫 화면으로 반영.
